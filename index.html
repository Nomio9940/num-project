Abstract

This thesis work report focuses on the application and implementation of Docker within the context of operating a school web system, specifically at the National University of Mongolia. Docker has emerged as a revolutionary tool for software deployment, providing a platform for developers to build, package, and distribute applications swiftly and reliably. This report outlines the significant advantages Docker can bring to the school's web system, including enhanced portability, scalability, and efficiency.

Introduction

Modern educational institutions rely heavily on web systems for various functions, ranging from administration and course content delivery to student engagement and assessment. Effective and efficient operation of these systems is crucial for an institution like the National University of Mongolia. Docker's deployment technology can play a vital role in improving the performance and reliability of these web systems.

Literature Review

Docker has been the subject of numerous studies, mainly focusing on its impact on software deployment, application isolation, and infrastructure optimization. Studies such as "Containerization and the PaaS Cloud" (Vaughan-Nichols, 2014) and "The Impact of Containerization on Microservice Architectures" (Newman, 2015) provide important context and foundation for our research.

Methodology

We conducted both quantitative and qualitative research to gather data for this report. A survey of the current school web system and a comparative study of Docker's implementation in similar settings were carried out. We also performed a detailed technical analysis of Docker's features to understand its potential impact on the school's web system.

Docker: An Overview

Docker is an open-source platform that simplifies the creation, deployment, and running of applications by using containerization. Docker containers package an application with everything it needs to run, including libraries, system tools, code, and runtime, ensuring consistency and reducing overhead.

Advantages of Docker

Docker's advantages include portability, scalability, isolation, version control, and resource optimization. These benefits can significantly improve the school web system's performance, reliability, and efficiency, leading to better user experience and satisfaction.

Docker Implementation: Case Study

A practical application of Docker was examined through a case study at a comparable institution, which reported increased system performance, lower resource usage, and streamlined deployment processes. This case study provides a blueprint for potential implementation at the National University of Mongolia.

Docker Integration in the National University of Mongolia

The National University of Mongolia's current web system can greatly benefit from Docker's containerization. Proposed steps for implementation include system assessment, Docker installation, container configuration, testing, and deployment.

Challenges and Solutions

While Docker promises numerous benefits, several challenges could arise during its implementation, such as legacy system compatibility, staff training, and security concerns. However, these challenges can be mitigated with thorough planning, continuous learning, and by leveraging Docker's robust community support.

Conclusion

The implementation of Docker at the National University of Mongolia holds substantial promise for enhancing the efficiency, scalability, and reliability of the school's web system. By utilizing Docker's powerful containerization technology, the University can significantly improve its web system's operational effectiveness and user experience.
